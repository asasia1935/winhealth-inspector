# Safety and Privacy

> 이 문서는 진단 및 관리 기능을 구현할 때 반드시 지켜야 할 안전 원칙을 정의합니다.

## 기본 원칙

WinHealth Inspector는 사용자의 명시적인 동의 없이 파일을 삭제하거나 Windows 설정을 변경하지 않는다.

진단 결과가 존재한다고 해서 관리 작업을 자동 실행하지 않는다.

## 진단 안전 원칙

- 검사 실패를 정상으로 표시하지 않는다.
- 검사 실패를 시스템 위험으로 표시하지 않는다.
- 지원하지 않는 환경은 `NotApplicable`로 처리한다.
- 권한 부족, 시간 초과, API 오류는 `Unknown` 또는 별도 실행 상태로 처리한다.
- 판정 기준이 불확실한 경우 단정적인 표현을 피한다.
- 다른 보안 제품이나 사용자 설정 가능성을 고려한다.

## 파일 정리 원칙

- 삭제 전 예상 파일 수와 용량을 표시한다.
- 사용자가 삭제 대상을 확인할 수 있어야 한다.
- 사용자 문서, 바탕 화면, 다운로드 폴더는 자동 정리 대상에 포함하지 않는다.
- 최근 사용된 임시 파일은 기본적으로 제외한다.
- 사용 중인 파일은 건너뛴다.
- 허용된 경로 밖의 파일은 삭제하지 않는다.
- 폴더 전체를 조건 없이 삭제하지 않는다.
- 심볼릭 링크 및 재분석 지점은 기본적으로 탐색하지 않는다.
- 삭제 실패는 숨기지 않고 결과에 표시한다.
- 부분 성공을 전체 성공으로 표시하지 않는다.

## 허용 경로 정책

관리 작업은 명시적으로 허용된 경로에서만 실행한다.

경로 비교 시 다음을 고려한다.

- 정규화된 절대 경로 사용
- 상위 경로 이탈 방지
- 대소문자 처리
- 심볼릭 링크 및 재분석 지점 확인
- 환경 변수 확장 후 검증
- 존재하지 않는 경로 처리

## 되돌릴 수 없는 작업

다음 작업은 실행 전에 별도로 경고한다.

- 휴지통 비우기
- 파일 영구 삭제
- 복구 기능을 제공하지 않는 관리 작업

되돌릴 수 없는 작업은 전체 최적화나 일괄 실행에 기본 포함하지 않는다.

## DNS 캐시 초기화

- DNS 서버 자체를 변경하지 않는다.
- TCP 설정이나 네트워크 레지스트리를 변경하지 않는다.
- 네트워크 진단 결과와 연결하여 제공한다.
- 관리자 권한이 필요한 경우 이를 명확히 표시한다.
- 실행 후 관련 진단을 다시 수행한다.

## 관리자 권한

- 애플리케이션 전체를 항상 관리자 권한으로 실행하지 않는다.
- 관리자 권한이 필요한 작업은 MVP에서 제외하거나 실행 불가 상태로 처리한다.
- 향후 관리자 작업을 지원할 경우 별도의 제한된 Helper 프로세스를 검토한다.
- Helper가 도입되더라도 허용 목록에 등록된 작업만 수행한다.

## 개인정보

- 로그인 기능을 제공하지 않는다.
- 사용자의 명시적인 동의 없이 앱 사용 정보, 오류 정보 또는 시스템 정보를 외부로 전송하지 않는다.
- 향후 선택적 오류 보고 기능을 제공하더라도 기본값은 비활성화하며, 전송 내용을 사용자가 확인할 수 있어야 한다.
- 진단 보고서 내보내기는 사용자의 명시적인 요청으로만 수행한다.
- 보고서에는 사용자 이름, 전체 파일 경로, 네트워크 식별 정보 등 민감할 수 있는 정보를 최소화한다.
- 원본 이벤트 로그 전체를 보고서에 그대로 포함하지 않는다.
- 로그와 보고서는 사용자가 삭제할 수 있어야 한다.

## 진단 표현

다음을 명확히 구분한다.

- 시스템이 정상임
- 참고할 정보가 있음
- 주의가 필요한 상태가 있음
- 비교적 빠른 조치가 필요한 상태가 있음
- 검사를 수행하지 못함
- 현재 장치에는 해당 검사가 적용되지 않음

상태는 색상만으로 표현하지 않고 아이콘과 텍스트를 함께 사용한다.

## 효과 표현

- 저장 공간 확보와 성능 향상을 구분한다.
- 임시 파일 삭제가 성능 향상을 보장한다고 표현하지 않는다.
- DNS 캐시 초기화가 인터넷 속도를 항상 개선한다고 표현하지 않는다.
- 시작 프로그램 수가 많다는 이유만으로 성능 문제라고 단정하지 않는다.
- 실제 측정 없이 보안 위험이나 시스템 손상을 과장하지 않는다.

## 제외할 위험 기능

MVP 및 기본 제품 방향에서 다음 기능을 제외한다.

- 레지스트리 일괄 삭제
- Windows 서비스 대량 비활성화
- TCP 및 네트워크 설정 자동 튜닝
- DNS 서버 강제 변경
- 메모리 강제 정리
- 페이지 파일 임의 변경
- 전원 계획 강제 변경
- CPU 우선순위 영구 변경
- 사용자 데이터 자동 삭제
- 브라우저 쿠키 및 로그인 정보 일괄 삭제
- 모든 임시 폴더의 무차별 삭제
- 문제 개수 부풀리기

## 외부 명령 및 프로세스 실행

- 실행 가능한 명령과 인자는 애플리케이션이 정의한 허용 목록으로 제한한다.
- 사용자 입력을 명령 문자열이나 셸 스크립트에 직접 결합하지 않는다.
- 가능하면 셸을 통하지 않고 실행 파일과 인자를 분리하여 전달한다.
- 실행 결과, 종료 코드 및 실패 이유를 기록한다.
- 관리자 권한이 필요한 명령은 사용자의 명시적인 선택 없이 실행하지 않는다.
