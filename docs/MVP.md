# MVP Scope

> 이 문서는 첫 공개 버전의 범위를 제한하기 위한 초안입니다. 구현 검증 결과에 따라 세부 항목은 조정할 수 있지만, 범위 확대는 별도 결정으로 다룹니다.

## MVP 목표

관리자 권한을 기본적으로 요구하지 않고, Windows의 주요 상태를 진단하며, 사용자가 선택한 안전한 관리 작업을 수행할 수 있는 첫 공개 버전을 만든다.

## MVP 완료 기준

MVP는 다음 흐름이 실제 PC에서 안정적으로 동작할 때 완료로 본다.

```text
전체 진단 실행
→ 결과 요약 및 상세 근거 확인
→ 관리 작업 미리보기
→ 사용자 확인 후 실행
→ 작업 결과 확인
→ 관련 진단 재실행
→ 작업 전후 결과 비교
```

## 진단 기능

### 1. 저장 공간

#### 검사 항목

- 고정 드라이브 목록
- 전체 용량
- 사용 가능 용량
- 남은 공간 비율
- 사용자 임시 폴더 예상 용량

#### 결과에 포함할 내용

- 드라이브별 측정값
- 정상·주의·위험 판정
- 판정 기준
- 저장 공간 부족 시 예상 영향
- Windows 저장소 설정 열기

#### 참고

정확한 정상·주의·위험 수치는 구현 전에 별도 기준으로 확정한다. 비율과 절대 용량을 함께 고려한다.

### 2. 최근 프로그램 및 시스템 오류

#### 검사 항목

- 최근 7일간 응용 프로그램 오류
- 최근 7일간 주요 시스템 오류
- 동일 오류 그룹화
- 마지막 발생 시간
- 반복 발생 횟수

#### 결과에 포함할 내용

- 오류가 발생한 프로그램 또는 공급자
- 이벤트 식별 정보
- 반복 횟수
- 마지막 발생 시각
- 권장 확인 사항

#### 제한

모든 이벤트를 문제로 판단하지 않는다. MVP에서는 반복 발생하거나 사용자 경험에 영향을 줄 가능성이 높은 오류를 우선적으로 다룬다.

### 3. Windows 보안 상태

#### 검사 항목

- Windows Defender 활성화 여부
- 실시간 보호 상태
- Windows 방화벽 상태
- 관련 주요 서비스 상태

#### 결과에 포함할 내용

- 확인된 상태
- 확인하지 못한 항목
- 다른 보안 프로그램 사용 가능성에 대한 안내
- Windows 보안 설정 열기

#### 제한

다른 백신을 완벽하게 판별하지 못한 상태에서 Defender 비활성화를 무조건 위험으로 단정하지 않는다.

### 4. Windows 업데이트 및 재부팅 상태

#### 검사 항목

- Windows Update 관련 서비스 상태
- 최근 설치된 업데이트 정보
- 재부팅 필요 여부
- 장기간 재부팅하지 않은 상태

#### 결과에 포함할 내용

- 최근 업데이트 정보
- 재부팅 필요 여부
- Windows 업데이트 설정 열기

#### 제한

MVP에서는 업데이트 설치를 직접 수행하지 않는다.

### 5. 시작 프로그램

#### 검사 항목

- 주요 시작 프로그램 목록
- 프로그램 이름
- 실행 파일 경로
- 등록 위치
- 존재하지 않는 실행 파일 경로

#### 결과에 포함할 내용

- 시작 프로그램 개수
- 등록 위치
- 파일 존재 여부
- Windows 시작 앱 설정 열기

#### 제한

시작 프로그램 수만으로 성능 문제를 단정하지 않는다. MVP에서는 직접 비활성화하거나 레지스트리 값을 변경하지 않는다.

### 6. 네트워크 기본 상태

#### 검사 항목

- 활성 네트워크 어댑터
- IP 주소 할당 여부
- 기본 게이트웨이 존재 여부
- DNS 서버 설정 여부
- DNS 이름 확인 성공 여부
- 인터넷 연결 가능 여부

#### 결과에 포함할 내용

- 어댑터 상태
- IP, 게이트웨이, DNS 단계별 결과
- 실패 지점
- Windows 네트워크 설정 열기

#### 제한

MVP에서는 DNS 서버 변경, TCP 설정 변경, 네트워크 초기화를 수행하지 않는다.

## 관리 작업

### 1. 오래된 사용자 임시 파일 선택 정리

#### 대상

- 현재 사용자의 임시 폴더
- 앱이 명시적으로 허용한 사용자 영역의 임시 경로

#### 필수 기능

- 삭제 예상 파일 수 표시
- 삭제 예상 용량 표시
- 삭제 대상 미리보기
- 최근 사용 파일 제외
- 사용 중인 파일 건너뛰기
- 사용자가 선택한 뒤 실행
- 성공, 일부 성공, 실패 결과 표시
- 정리 후 저장 공간 진단 재실행

#### 안전 제한

- 사용자 문서, 바탕 화면, 다운로드 폴더를 포함하지 않는다.
- 허용된 경로 밖의 파일을 삭제하지 않는다.
- 심볼릭 링크 및 재분석 지점은 기본적으로 탐색하지 않는다.
- 삭제 실패를 숨기지 않는다.
- 폴더 전체를 무조건 삭제하지 않는다.

### 2. 휴지통 용량 확인 및 명시적 비우기

#### 필수 기능

- 현재 휴지통 사용량 표시
- 되돌릴 수 없는 작업임을 안내
- 다른 작업과 분리된 명시적 확인
- 실행 결과 표시
- 정리 후 저장 공간 진단 재실행

#### 제한

전체 최적화 같은 일괄 작업에 자동 포함하지 않는다.

### 3. DNS 캐시 초기화

#### 제공 조건

- 네트워크 진단 결과와 연결한다.
- DNS 이름 확인 실패 또는 관련 문제 상황에서 사용자가 선택할 수 있도록 한다.

#### 필수 기능

- 실행 전 영향 설명
- 실행 결과 표시
- 실행 후 DNS 이름 확인 재실행
- 작업 전후 결과 비교

#### 제한

관리자 권한이 필요한 환경에서는 실행할 수 없음 또는 권한 필요 상태로 처리한다. MVP에서는 별도 관리자 Helper를 구현하지 않는다.

## 상태 정의

### 진단 상태

- `Healthy`: 권장 기준 안에 있음
- `Information`: 문제는 아니지만 참고할 내용이 있음
- `Warning`: 확인하거나 조치하는 것이 좋음
- `Critical`: 비교적 빠른 확인 또는 조치가 필요함
- `Unknown`: 오류, 권한 부족, 시간 초과 등으로 확인하지 못함
- `NotApplicable`: 현재 장치나 환경에 적용되지 않음

### 관리 작업 상태

- `Ready`: 실행 가능
- `ConfirmationRequired`: 별도 확인 필요
- `RequiresAdministrator`: 관리자 권한 필요
- `Running`: 실행 중
- `Succeeded`: 성공
- `PartiallySucceeded`: 일부 성공
- `Failed`: 실패
- `Cancelled`: 취소됨
- `RestartRequired`: 재부팅 필요
